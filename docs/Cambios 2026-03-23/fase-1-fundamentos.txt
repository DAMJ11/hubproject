2. MIGRACIÓN DE COLORES HARDCODEADOS → BRAND TOKENS
================================================================================

  MAPEO DE REEMPLAZOS:
    #2563eb  (blue-600)    →  brand-600  (#4f46e5 Indigo)
    #1d4ed8  (blue-700)    →  brand-700  (#4338ca Indigo)
    #111827  (gray-900)    →  brand-900  (#1e1b4b Indigo)
    #1a365d  (dark blue)   →  brand-900  (#1e1b4b Indigo)
    #1f2937  (gray-800)    →  gray-800   (Tailwind neutral)

  CLASES TAILWIND MIGRADAS (ejemplos):
    bg-[#2563eb]           →  bg-brand-600
    hover:bg-[#1d4ed8]     →  hover:bg-brand-700
    text-[#2563eb]         →  text-brand-600
    border-[#2563eb]       →  border-brand-600
    focus:ring-[#2563eb]   →  focus:ring-brand-600
    bg-[#2563eb]/10        →  bg-brand-100
    bg-[#111827]           →  bg-brand-900
    text-[#1f2937]         →  text-gray-800

  SVG fills (para-fabricantes):
    #2563eb  →  #4f46e5  (Indigo 600 directo, necesario en atributo fill)
    #1a365d  →  #1e1b4b  (Indigo 900 directo)

4. ANIMACIONES — CONSOLIDACIÓN
================================================================================

  ANTES:
    - 3 sistemas de animación compitiendo:
      1. src/styles/transitions.css (wildcard * con transition en TODO)
      2. src/components/ui/page-transition.tsx (Framer Motion)
      3. tailwindcss-animate
    - Wildcard * selector con transition-property afectaba rendimiento
    - Utilidades duplicadas (.card-hover, .btn-hover-lift, etc.)

  DESPUÉS:
    - transitions.css ELIMINADO
    - Keyframes útiles consolidados en globals.css:
      fadeInUp, fadeInScale, slideInRight
    - Clases de utilidad preservadas:
      .animate-fadeInUp, .animate-fadeInScale, .animate-slideInRight
    - scroll-behavior: smooth preservado
    - @media (prefers-reduced-motion: reduce) preservado
    - Wildcard transition-property ELIMINADO (mejora rendimiento)

  ARCHIVOS ELIMINADOS:
    - src/styles/transitions.css

  ARCHIVOS MODIFICADOS:
    - src/app/globals.css              → Keyframes y utilidades consolidados
