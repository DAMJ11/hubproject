CAMBIOS REALIZADOS
-------------------

  1. COMPONENTE BADGE (shadcn)
     - Creado: src/components/ui/badge.tsx
     - Variantes: default, secondary, destructive, outline
     - Usa class-variance-authority (mismo patrón que button.tsx)

  2. TOASTS EN CAPABILITIES
     - Archivo: src/app/[locale]/dashboard/company/capabilities/page.tsx
     - handleAdd: toast.success/error en éxito/fallo
     - handleDelete: toast.success/error con verificación de res.ok
     - Traducciones: addSuccess/addError/deleteSuccess/deleteError (es/en/fr)

  3. DYNAMIC IMPORT — MESSAGESPANEL
     - Archivo: src/app/[locale]/dashboard/messages/page.tsx
     - Cambiado: import estático via barrel → dynamic(() => import(...))
     - Loading: reutiliza MessagesLoading como fallback
     - Beneficio: ~50KB de Pusher diferido hasta que el usuario entre a mensajes

  4. ARIA-LABELS (ACCESIBILIDAD)
     - capabilities: botón delete → aria-label={t("delete")}
     - certifications: botón delete → aria-label={t("delete")}
     - settings: toggle contraseña → aria-label showPassword/hidePassword
     - rfq-form: botón back + botón remove material
     - Traducciones en Capabilities, Certifications, Settings, ProjectNew (es/en/fr)

  5. MIGRACIÓN A SERVER COMPONENTS (8 páginas)

     reviews/page.tsx → Server Component
       - Data: lib/data/reviews.ts → getReviewsList()
       - Client: reviews/reviews-list.tsx (filtros + listado)

     company/page.tsx → Server Component
       - Data: lib/data/companies.ts → getCurrentCompany()
       - Client: company/company-form.tsx (edición de empresa)

     company/capabilities/page.tsx → Server Component
       - Data: lib/data/capabilities.ts → getCapabilitiesWithCategories()
       - Client: capabilities/capabilities-manager.tsx (CRUD capacidades)

     company/certifications/page.tsx → Server Component
       - Data: lib/data/certifications.ts → getCertifications()
       - Client: certifications/certifications-manager.tsx (CRUD certs)

     contracts/[id]/page.tsx → Server Component
       - Data: lib/data/contracts.ts → getContractDetail()
       - Client: contracts/[id]/contract-detail.tsx (timeline + acciones)

     manufacturers/page.tsx → Server Component
       - Data: query() directa para categorías
       - Client: manufacturers/manufacturers-list.tsx (filtros + grid)

     settings/page.tsx → Server Component
       - Data: lib/data/settings.ts → getSettingsUser()
       - Client: settings/settings-form.tsx (perfil + seguridad + notifs)

     addresses/page.tsx → Server Component
       - Data: lib/data/addresses.ts → getAddresses()
       - Client: addresses/addresses-manager.tsx (CRUD direcciones)
