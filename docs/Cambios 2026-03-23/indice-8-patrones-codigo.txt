8.2 PATRÓN SERVER/CLIENT COMPONENT
====================================

4 páginas migradas al patrón: Server Component (page.tsx) que obtiene
datos en el servidor → Client Leaf Component que recibe props y maneja
interactividad (filtros, localStorage, etc.)

  1. projects/page.tsx → projects-list.tsx
     - Data: getRfqList() desde lib/data/rfq.ts
     - Props: projects: RFQItem[]

  2. manufacturers/page.tsx → manufacturers-list.tsx
     - Data: getCurrentUser() + query categories
     - Props: userRole: string, categories: Category[]
     - Nota: manufacturers sigue fetch dinámico por paginación/debounce

  3. reviews/page.tsx → reviews-list.tsx
     - Data: getReviewsList() desde lib/data/reviews.ts (NUEVO)
     - Props: reviews: ReviewItem[], isAdmin: boolean

  4. payments/page.tsx → payments-list.tsx
     - Data: getPaymentsList() desde lib/data/payments.ts (NUEVO)
     - Props: payments: PaymentItem[], totals: PaymentTotals

Nuevas funciones de datos (server-only):
  - src/lib/data/reviews.ts  → getReviewsList()
  - src/lib/data/payments.ts → getPaymentsList()
