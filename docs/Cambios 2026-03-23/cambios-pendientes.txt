SECCIÓN 1 — DEUDA TÉCNICA IDENTIFICADA
────────────────────────────────────────

  1.1 NAMING CONVENTIONS (21+ archivos)
  ──────────────────────────────────────
  21 archivos de componentes usan kebab-case en vez de PascalCase.
  4 archivos en lib/ usan kebab-case en vez de camelCase.
  No se renombraron porque romperían 40+ import paths en toda la app.
  Requiere una sesión dedicada de refactoring con herramienta de renombrado.

  Archivos afectados (componentes):
    - empty-state.tsx → EmptyState.tsx
    - skeleton-loader.tsx → SkeletonLoader.tsx
    - status-badge.tsx → StatusBadge.tsx
    - page-transition.tsx → PageTransition.tsx
    - dropdown-menu.tsx, sheet.tsx, switch.tsx, etc. (shadcn, exentos)

  Archivos afectados (lib):
    - green-score.ts → greenScore.ts
    - status-config.ts → statusConfig.ts

  1.2 CLAVES i18n INCONSISTENTES (3 violaciones)
  ────────────────────────────────────────────────
  - "green-score" → debería ser "greenScore"
  - "in_production" → debería ser "inProduction" (2 ocurrencias)
  Cambiarlas requiere actualizar los 3 archivos de idioma + componentes
  que las consumen.

  1.3 COLORES HEX RESIDUALES (~60 en 2 archivos)
  ────────────────────────────────────────────────
  PlatformSection.tsx y QualitySection.tsx conservan ~60 colores hex únicos.
  Son gradientes decorativos de cards donde cada color es de un solo uso.
  Crear CSS vars para colores únicos añade complejidad sin beneficio.
  Decisión: Aceptable como excepción documentada.
